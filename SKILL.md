# ⚡ 执行指令（AI 绝对遵守）

> **⚠️ 核心铁律（违反此条 IRY 就会失效）：**
> **AI 每输出一条信息/问题后，必须立即 STOP，等待用户回复。**
> 用户回复后才能开始下一步。
> **严禁**：一次性输出多个问题 / 跳过追问直接生成 / 在用户回复前做任何事。

**正确的行为**：
```
IRY → 输出「确认启动？」→ ⏸️ STOP → 用户说「是」
→ Step B：确认项目（Q1 名称 + Q2 路径）→ ⏸️ STOP → 用户回复
→ Step B3：文件读取授权确认（A/B/C）→ ⏸️ STOP → 用户选择
→ Step B4：汇总确认 → ⏸️ STOP → 用户核对
→ Step C：浏览分析（含进度条）→ ⏸️ STOP → 用户确认
→ Step D：选择文件详细度 → ⏸️ STOP → 用户选择
→ Step E：AI 拉清单 → 用户核对 → 有遗漏补扫
→ Step F：生成记忆包（10 文件）→ ⏸️ STOP → 等用户确认
→ Step G：交付 → ⏸️ STOP → 反馈收集
```

**进度条规则**：每个 Step 开始时，AI 必须输出 `进度：Step X/7，当前：Step X - {状态描述}...`

# 🎯 交接目标

本交接包的定位是：**让一个对该项目完全一无所知的智能体（接收AI），
通过阅读本包内的全部文件，瞬间继承移交AI 对该项目的全部了解，
进入准工作状态，实现无缝衔接。**

接收AI 阅读完成后，应能做到：
- **无需追问**即可回答关于项目的基础问题
- 理解项目的**业务背景**和**技术背景**
- 知晓**当前进度**和**下一步优先级**
- 清楚哪些事**能做**、哪些事**不能做**（红线）
- 知道**出问题了找谁、查哪里**

### F.2 文件详细度（统一按 Deep 深度生成）

# 📝 通用规则（贯穿全流程）

1. **单步执行**：每次只输出一个 Step / 一个问题，输出后 ⏸️ STOP
2. **进度条**：每 Step 开始时输出 `进度：Step X/7`
3. **浏览进度**：大型项目分步输出 `[1/3] [2/3] [3/3]`
4. **取消选项**：每步必须有取消/回退选项
5. **错误处理**：任何步骤失败 → 提示用户 → 等用户决策
6. **文件编码**：所有交付包文件强制 UTF-8
7. **时间戳**：所有生成文件必须在文件头标注生成时间
8. **暂缺标注**：缺失内容写 `[暂缺]`，不编造
9. **反馈机制**：交付后引导用户记录 feedback
10. **跨会话续接**：每次触发都从头开始（不保留上次进度）

# 🎯 产品理念

IRY = **I Remember You**（我记得你）

这不是工具名，是一句承诺：**当你换了新的 AI 助手，它依然记得你和老 AI 之间的一切。**

就像换了手机，联系人、照片、聊天记录全部同步过来——IRY 做的就是这件事，只不过同步的对象是 **AI 的记忆**。

---

| 能力 | 说明 |
|------|------|
| 用户画像转移 | 沟通风格、表达习惯、偏好设置 |
| 项目状态快照 | 当前任务进度、待办事项、阻塞点 |
| 经验库迁移 | 踩过的坑、解决方案、最佳实践 |
| 决策记录 | 关键决策、选型理由、历史路线 |
| 雷区预警 | 绝对不能做的事、历史错误记录 |
| 无缝衔接 | 接收AI 无需追问即可直接开工 |
