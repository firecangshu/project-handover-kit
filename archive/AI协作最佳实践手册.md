## 第一章：Prompt 工程基础

### 1.1 好 Prompt 的四个要素

一个高质量的 prompt，应该包含：

1. **角色设定**：告诉 AI 它是谁、具备什么能力
2. **背景信息**：项目背景、目标受众、约束条件
3. **任务要求**：具体要做什么、输出格式、质量标准
4. **示例参考**（可选）：给 1-2 个例子，AI 输出会更准

**反面例子：**
```
帮我写个文案
```

**正面例子：**
```
你是一位有 10 年经验的文旅品牌文案策划，擅长用故事感打动年轻用户。

【背景】
我们是一个做城市微度假的文旅品牌，目标用户是 25-35 岁的都市白领，主打"周末 2 天逃离城市"的概念。

【任务】
帮我写一篇小红书推广文案，主题是"周末去山里住一晚"。
要求：
1. 开头要有钩子，3 秒抓住注意力
2. 用第一人称，有真实感
3. 字数 500-800 字
4. 结尾要有互动引导

【参考风格】
类似"我在山里住了一晚，治好了我的精神内耗"这种感觉。
```

### 1.2 常用 Prompt 结构模板

**万能公式：**
```
你是【角色】，擅长【能力】。

【背景信息】
...

【任务要求】
...

【输出格式】
...

【约束条件】
...
```

### 1.3 让 AI 输出更稳定的技巧

1. **给标准，不给感觉**：不说"写得好一点"，说"符合 XX 行业规范，通过率 90% 以上"
2. **给例子，不给描述**：不说"活泼一点"，直接给 1-2 段你觉得活泼的文字
3. **分步拆解**：复杂任务拆成几步，让 AI 一步步做，比一次性全给效果好
4. **让 AI 自检**：输出后让 AI 自己检查"有没有符合所有要求"，准确率会提升

## 第二章：上下文管理

### 2.1 为什么上下文会丢？

AI 的上下文是有限的。会话太长、换了会话、换了模型，之前的信息就没了。

**常见的上下文丢失场景：**
- 会话过期/被重置
- 切换到新的 AI 模型
- 项目交接给同事/另一个 AI
- 隔了一段时间再回来做项目

### 2.2 上下文管理的三个层次

**第一层：会话内管理**
- 重要决策、规则变更，明确说"记住这个，后面都要遵守"
- 定期让 AI 总结当前进度和关键信息
- 复杂项目分会话，每个会话聚焦一个主题

**第二层：跨会话迁移**
- 用 IRY 生成记忆包，一键迁移到新会话
- 关键文档和决策记录保存到本地，随时可以喂给新 AI

**第三层：长期知识库**
- 建立个人/团队的 AI 协作知识库
- 沉淀常用 prompt 模板、行业知识、项目模板
- 新 AI 上来先喂知识库，再开始干活

### 2.3 哪些信息最值得保存？

不是所有信息都要保存，重点保存这几类：

1. **用户画像**：你的偏好、风格、工作习惯、禁忌要求
2. **项目背景**：项目目标、受众、约束条件、关键决策
3. **技术/业务规则**：格式要求、质量标准、不能碰的红线
4. **历史经验**：踩过的坑、验证过的好方法、失败案例
5. **当前状态**：做到哪了、下一步是什么、有什么阻塞

**这正好对应 IRY 的 10 个交接文件。**

## 第三章：IRY 工作流

### 3.1 什么时候用 IRY？

**高频场景：**
- 会话快过期了，先导出记忆包，开新会话导入
- 想换个模型试试，把上下文带过去
- 项目交接给同事，连 AI 记忆一起交
- 长期项目，定期导出记忆包做备份
- 一个项目多个 AI 分工，用记忆包同步信息

### 3.2 IRY 使用最佳实践

1. **定期导出**：不要等会话快过期了才想起来，养成"重要节点导出一次"的习惯
2. **补充备注**：IRY 扫描不到的隐性知识（比如你和甲方的私下约定），手动补充到备注里
3. **接收时校验**：新 AI 导入记忆包后，先让它复述一下"你理解的项目情况"，确认有没有偏差
4. **迭代更新**：项目有重大变更后，重新导出一次记忆包，保持最新

### 3.3 记忆包怎么用？

**方法一：直接喂给新 AI**
把记忆包文件内容直接粘贴到新会话开头，说"这是项目的全部背景，你先了解一下，然后我们开始工作"。

**方法二：分文件按需导入**
不需要全部导入的时候，挑相关的文件用。比如只需要技术背景，就只导入 02-技术规格.md。

**方法三：作为项目文档**
记忆包本身就是一份结构化的项目文档，可以直接给人看，也可以作为项目交接的正式材料。

## 第四章：场景化最佳实践

### 4.1 开发者场景

**痛点：**
- 换会话后，AI 忘了项目的技术栈、代码规范、架构设计
- 调试了半天的 bug，新会话又得重新描述一遍

**最佳实践：**
1. 项目初期就用 IRY 导出一次基础记忆包
2. 每次重大架构决策后，更新一次记忆包
3. 调试 bug 前，先把相关代码和错误信息喂给 AI，再加上记忆包
4. 用 IRY 的 05-已知问题.md 记录踩过的坑，避免重复踩

**推荐文件组合：**
- 00-项目概况.md
- 02-技术规格.md
- 06-核心文件索引.md
- 07-环境配置.md
- 08-决策记录.md
- 05-已知问题.md

### 4.2 内容创作者场景

**痛点：**
- 换会话后，AI 忘了你的写作风格、品牌调性、目标受众
- 之前定好的选题方向、爆款模板，新 AI 全忘了

**最佳实践：**
1. 把你的风格偏好、爆款案例、选题方向整理好，用 IRY 生成"创作者记忆包"
2. 每个新项目/新系列，单独导出一个记忆包
3. 换 AI 模型时，先导入记忆包，再让它写 1-2 篇试手，校准风格
4. 用 IRY 的 01-用户画像.md 详细记录你的偏好，越细越好

**推荐文件组合：**
- 00-项目概况.md
- 01-用户画像.md（重点写风格偏好）
- 03-进度计划.md（选题排期）
- 08-决策记录.md（为什么选这个方向、哪些标题爆过）

### 4.3 产品/策划场景

**痛点：**
- 换会话后，AI 忘了产品定位、用户画像、竞品情况
- 之前讨论过的方案、被否决的想法，又被提出来一遍

**最佳实践：**
1. 项目立项时就导出完整记忆包
2. 每次重要讨论后，更新决策记录
3. 用 IRY 的 04-依赖与约束.md 记录所有限制条件（预算、时间、技术限制等）
4. 方案评审前，先让 AI 基于记忆包做准备，效率更高

**推荐文件组合：**
- 00-项目概况.md
- 03-进度计划.md
- 04-依赖与约束.md
- 08-决策记录.md
- 05-已知问题.md

## 第五章：常见问题与避坑指南

### 5.1 AI 输出质量不稳定怎么办？

**原因：** 上下文不够、prompt 不清晰、模型本身的随机性

**解决方法：**
1. 给更详细的背景和要求
2. 给具体的例子和标准
3. 让 AI 分步输出，每步确认后再继续
4. 重要内容让 AI 输出 2-3 个版本，你选最好的

### 5.2 上下文太长，AI 记不住前面的怎么办？

**原因：** AI 的上下文窗口有限，太长了前面的信息会被"遗忘"

**解决方法：**
1. 定期让 AI 总结关键信息，用总结替代原始对话
2. 重要规则和决策，隔一段时间重复强调一次
3. 用 IRY 导出记忆包，开新会话重新导入（相当于"刷新"上下文）
4. 非核心信息放到文档里，需要时再引用

### 5.3 换模型后效果差很多怎么办？

**原因：** 不同模型的能力、风格、理解方式都不一样

**解决方法：**
1. 不要直接复制粘贴 prompt，根据模型特点调整
2. 先导入记忆包，再让新模型"热身"——做几个简单任务校准风格
3. 重要项目，固定用一个模型，不要频繁换
4. 用 IRY 记录每个模型的特点和适配场景，下次直接参考

### 5.4 IRY 扫描出来的信息不全怎么办？

**正常的。** IRY 是基于文件扫描的，扫描不到的隐性知识需要你补充。

**补充方法：**
1. 在 Step E 的自适应提问环节，把补充信息写到备注里
2. 生成记忆包后，手动编辑缺失的文件，补充你觉得重要的信息
3. 重要的隐性知识，直接写到项目文档里，下次 IRY 就能扫描到了

**记住：IRY 是帮你省 80% 的力气，剩下 20% 的细节还是要你把关。**

## 附录：常用 Prompt 模板

### A.1 让 AI 学习你的风格

```
我会给你几段我写的文字，你仔细学习我的写作风格、语气、常用表达。
学完后告诉我你总结出了哪些特点，然后我们再开始工作。

【我的文字】
...（粘贴 3-5 段你写的内容）
```

### A.2 让 AI 复述确认

```
你先复述一下你理解的项目情况和我的要求，确认没问题了我们再开始。
```

### A.3 让 AI 分步输出

```
这个任务我们分几步做：
第一步：先...
第二步：再...
第三步：最后...

我们先做第一步，做完确认后再继续。
```

### A.4 导入记忆包后的开场

```
这是这个项目的完整背景和历史记录，你先仔细读一遍。
读完后用 3-5 句话总结一下：这是什么项目、目前做到哪了、下一步要做什么。
确认没问题了我们再开始工作。

【记忆包内容】
...（粘贴 IRY 生成的记忆包）
```

### A.5 让 AI 自我检查

```
输出完后，你自己检查一遍：
1. 有没有符合所有要求？
2. 有没有遗漏什么？
3. 有没有可以优化的地方？

有问题就自己改，没问题就说"确认完成"。
```
