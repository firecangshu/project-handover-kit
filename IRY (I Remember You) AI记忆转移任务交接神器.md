# IRY (我记得你) AI→AI任务无缝交接神器

> **核心价值**：将移交AI 的上下文（用户画像、项目状态、经验、雷区）无损迁移至接收AI，通过人机协同与质检循环，确保接收AI 无需追问即可开工。
>
> **目标用户**：AI 编程、程序员、软件工程师
>
> **解决痛点**：信息转移时的不对称，免除因缺项/漏项导致的交接不顺畅、不全面
>
> **核心承诺**：**无脑交接，无缝衔接，免去后顾之忧**

---

## 核心设计原则

1. **傻瓜式交互**：用户全程只需做选择题（A/B/C）或简短补充，无需组织语言。
2. **强依赖关系**：后一步严格依赖前一步的输出，禁止跳跃。
3. **质检循环（QC Loop）**：清单检查不合格 → 自动回滚补充 → 再次检查，最多 3 次循环。
4. **人工抽检（Spot Check）**：交付给接收AI 后，用户自问 1 题验证接收AI 是否真读懂，**不限制次数**（相信用户判断）。
5. **权责分离**：
   - **移交AI**：Step 1-3，人机协同完成交接包后告一段落。
   - **人**：Step 1-3 确认、Step 4 传包、Step 5 测试打分。
   - **接收AI**：Step 4-5，读包深度学习、接收测试。

## 详细工作流程

### 全局规则

- **进度条**：每步开始时，AI 输出文字进度条：`进度：Step X/5，当前：Step X - {状态描述}...`
- **触发确认**：任何触发词命中后，AI 先弹窗确认「是否启动IRY (我记得你)？A. 是 / B. 否」，防止误触发
- **取消选项**：每步的 AskUserQuestion 都包含「取消交接」选项
- **取消清理**：用户选择取消后，已生成的文件由用户决定是否保留
- **错误处理**：任何步骤失败 → 输出错误信息 → 询问用户「重新生成？」OR「放弃交接程序」
- **跨会话续接**：每次触发都从头开始（不保留上次进度）
- **文件编码**：所有交付包文件强制 UTF-8
- **大文件处理**：单文件 > 10MB → 只记录路径不复制内容，额外生成 `大文件清单.txt`

### Step 1：人工确认（Human Gate）

**目标**：收集核心参数，每个答案都要落地（影响后面行为）。

---

#### Q1：任务确认

**目的**：确认任务身份，决定后续搜索范围。

**项目名推断规则**：
1. 移交AI 优先从当前工作目录名推断项目名，展示给用户确认
2. 如果用户否定 -> AI 从目录内项目配置文件筛选最可能的项目名
3. 如果依然被否定 -> 用户手动输入项目名，并询问「是否需要同步该项目相关信息？」

| 选项 | 含义 | 落地 |
|------|------|------|
| A | 是，任务名 = {目录名推断 -> 用户确认} | Q3 默认 A（当前目录），可跳过 |
| B | 否，任务名 = [用户输入，含是否同步信息] | Q3 强制 B（指定目录） |
| C | 取消交接 | 终止 |

**项目名来源**：Q1 确认后的任务名，作为后续交付包命名的一部分。

---

#### Q2：隐私授权

**目的**：确定隐私内容处理策略，直接影响 Step 2 的搜索范围。

| 选项 | 含义 | 落地 |
|------|------|------|
| A | 完全授权 | Step 2 搜索**包含**隐私，生成文件时保留隐私内容 |
| B | 部分脱敏 | **追问** -> Q2.1 |
| C | 完全拒绝 | Step 2 搜索跳过隐私内容 |

**Q2.1（追问）**：「哪些内容需要脱敏？（可多选）」

| 选项 | 含义 | 落地 |
|------|------|------|
| A | 密码 / Token / API Key | 记录到脱敏清单.txt |
| B | 个人身份信息（姓名/电话/邮箱） | 记录到脱敏清单.txt |
| C | 业务逻辑（核心算法/商业机密） | 记录到脱敏清单.txt |
| D | 以上都不需要脱敏 | 不生成脱敏清单 |

**落地说明**：
- Q2 选 B -> 记录脱敏清单 -> Step 2 搜索包含隐私，但生成文件时**删除**脱敏清单中的内容（用 [已脱敏] 占位）
- 脱敏清单随交付包一并生成

#### Q3：扫描文件范围

**目的**：确定 Step 2 要扫描哪些文件。
**依赖**：Q1 答案自动设定默认值（Q1 选 A -> Q3 默认 A；Q1 选 B -> Q3 强制 B）。

| 选项 | 含义 | 落地 |
|------|------|------|
| A | 当前工作目录（Q1 选 A 时默认选中） | Step 2 扫描当前目录 |
| B | 指定目录（Q1 选 B 时强制选中） | **追问** -> "请输入目录路径" |
| C | 手动选择文件 | **追问** -> 弹窗选文件范围 |

**追问弹窗**（选 C 时）：
| 选项 | 含义 |
|------|------|
| A | 当前文件夹 |
| B | 其他文件夹（弹窗选路径） |
| C | 取消 |

### Step 2：AI 自动整理（对话中生成，不落盘）

**依赖**：AuthParams from Step 1。

**动作**：移交AI 在对话中生成交付包内容（8 个文件），**暂不写入磁盘**。
- 文件内容展示在对话中供用户预览
- 最终确认写入在 Step 4 一次性完成
- **进度反馈**：每生成 2-3 个文件输出一次进度（如「文件 3/8：03-State-Plan.md 已完成」）

**信息源分级**：

| 级别 | 条件 | 行为 |
|------|------|------|
| 自动 | 有 WorkBuddy 文件系统能力（如 Read/Glob/Grep） | AI 主动扫描工作目录获取信息 |
| 授权 | 需用户点击授权才能访问文件系统 | AI 提示用户授权后扫描 |
| 退级 | 当前平台无文件系统访问能力 | 用户手动提供目录路径或文件列表 |

**信息不确定性处理**：
- 如果移交AI 对某个信息不确定（如只记得 2 条雷区），必须声明「未知」并等待用户补充
- **禁止猜测或编造信息**
- 用户补充后，AI 更新到对应文件中

如果 AI 不确定某个信息，话术模板：「关于 {主题}，我目前掌握的信息不完整，请你补充确认？」

**输出**：Handoff_Files[]（8 个文件内容，对话中展示）

**生成顺序**（有依赖关系，按编号顺序生成）：
1. `00-启动指令.md`（固定文本，无需依赖）
2. `01-User-DNA.md`（凭记忆，先写，接收AI 优先读）
3. `02-Project-Spec.md`（扫描 + 记忆，需先确认项目名）
4. `03-State-Plan.md`（扫描 + 记忆，依赖项目路径）
5. `04-Constraints.md`（依赖 Q2 答案和 01-03 内容）
6. `05-Known-Bugs.md`（凭记忆，独立）
7. `06-File-Index.md`（需先扫描完毕所有文件）
8. `07-Setup.md`（扫描 + 记忆，最后收尾）

| # | 文件名 | 内容 | 来源 |
|---|--------|------|------|
| 0 | 00-启动指令.md | 接收AI 启动指令（固定文本） | 固定 |
| 1 | 01-User-DNA.md | 用户风格、纠错习惯、3 条雷区 | 凭记忆 |
| 2 | 02-Project-Spec.md | 项目名、本地路径、最终目的（DoD） | 扫描 + 记忆 |
| 3 | 03-State-Plan.md | 当前进度（文件:行号）、下一步任务 | 扫描 + 记忆 |
| 4 | 04-Constraints.md | 隐私处理方案（基于 Q2）、限制条件 | 凭记忆 + Q2 答案 |
| 5 | 05-Known-Bugs.md | 已知 Bug + 踩坑清单 | 凭记忆 |
| 6 | 06-File-Index.md | 核心文件索引 + 用途说明 | 扫描工作目录 |
| 7 | 07-Setup.md | 环境配置 + 依赖安装步骤 | 扫描 + 记忆 |

**取消交接**：本步开始时输出「是否继续？A. 继续 / B. 取消交接（已生成的文件由用户决定是否保留）」

### Step 3：清单校准（QC Loop）

**依赖**：Handoff_Files[] from Step 2。

**动作**：对话中展示清单，等待用户裁决。

**核心问题**：「清单是否完整？」

| 选项 | 含义 | 落地 |
|------|------|------|
| A | 是，完整 | 进入 Step 4 |
| B | 否，补充 | **追问** -> 给出选择题指定补充内容（不用填空） |
| C | 取消交接 | 终止 |

**分支逻辑**：

```
IF "A" (是):
  -> 进入 Step 4

IF "B" (否，补充):
  -> 1. qc_count += 1
  -> 2. IF qc_count >= 3:
         -> 输出：警告已循环 3 次，强制交付并标注风险。请人工复核。
         -> 强制进入 Step 4
  -> 3. ELSE:
         -> 追问：用 AskUserQuestion 给出补充选项（如：补充文件/补充说明/修改文件），用户选择后 AI 执行
         -> 重新进入 Step 3 展示清单
```

**追问示例（AskUserQuestion）**：

| 选项 | 含义 |
|------|------|
| A | 补充一个文件（指定文件名和内容） |
| B | 补充一段说明（指定主题） |
| C | 修改现有文件（指定文件名和修改内容） |
| D | 完成 |

**QC Loop 计数器**：qc_count 初始为 0，每次循环 +1。

**取消交接**：选项 C = 取消交接

---

### Step 4：最终交付（移交AI 收尾）

**依赖**：LockedPackage from Step 3。

**动作**：
1. 将对话中确认的 8 个文件**一次性写入磁盘**到指定交付地址
2. 生成 09-后一步操作说明.md 放入包内
3. 在对话中输出交付陈述，告知用户后续操作

**落盘兜底**：如果当前平台无文件系统写入权限（如 GPTs、Claude Projects），AI 改为在对话中输出所有文件内容，提示用户手动复制保存到本地。

**输出目录**：{交付地址}\{项目名}-IRY (我记得你)交接包-{执行时间}\

> 注意：目录名中的时间戳 = Step 4 执行时的当前时间（不是 Q4 的「信息基准时间」）。
> Q4 的基准时间只影响文件内部的内容标注，不影响目录名。

**对话输出**：

```
 交付包已生成并落盘！

 交付路径：{交付地址}\{项目名}-IRY (我记得你)交接包-{时间戳}\ 

 文件清单：
  +-- 00-启动指令.md（请先给接收AI 看这个）
  +-- 01-User-DNA.md
  +-- 02-Project-Spec.md
  +-- 03-State-Plan.md
  +-- 04-Constraints.md
  +-- 05-Known-Bugs.md
  +-- 06-File-Index.md
  +-- 07-Setup.md
  +-- 08-补充项.md（如果有）
  +-- 09-后一步操作说明.md

警告 移交AI 的工作已全部结束。

请阅读 09-后一步操作说明.md，按指引完成：
(1) 将本包传给接收AI
(2) 接收AI 深度学习
(3) 人工抽检测试
```

**取消交接**：选项 B = 取消交接（已生成的文件由用户决定是否保留）

---

### Step 5：人工抽检（人 -> 接收AI）

**注意**：本步与移交AI 无关，纯属用户测试接收AI 的接收能力。
用户已将交付包发给接收AI，接收AI 已输出 [VERIFICATION] 区块后。

**动作**：
1. 用户**自拟 1 道项目相关问题**（用户最清楚关键点）
2. 接收AI 回答
3. 用户用「红绿灯」判分表判断

**红绿灯判分表**：

| 检查项 | 通过（读懂了） | 打回（没读懂） |
|--------|-------------------|------------------|
| 定位 | 精确到文件:行号 | 泛泛而谈、答非所问 |
| 雷区 | 准确复述 3 条禁忌 | 模糊承诺、「我会注意」 |
| 任务 | 下一步动作具体、可验收 | 宏大叙事、「我会研究」 |

**打回原因三选一**：

```
[打回] 请选择原因：

A. 信息遗漏（缺少关键内容）
   -> 补充 08-补充项.md，然后回 Step 3 -> Step 4 -> Step 5

B. 表述不清（文件写得不够清楚，接收AI 没读懂）
   -> 标记问题文件 -> 修改该文件内容，然后回 Step 4 -> Step 5

C. 接收AI 没认真读（回答全是套话）
   -> 不修改文件，直接回到 Step 5，用户再问一题
```

**分支逻辑**：

```
IF 通过:
  -> 输出：「抽检通过。接收AI 可以开工。」
  -> 提醒：「如有问题，反馈到 feedback.md，下次交接会自动检查。」
  -> 任务完成

IF 打回:
  -> 先选原因（A / B / C，见上）
  -> 按对应逻辑回滚
  -> 回滚后重新进入对应步骤
  -> 此循环**不限制次数**（相信用户判断）
```
