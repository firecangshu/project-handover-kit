| 01-用户画像.md | ⭐⭐⭐ | 相关人员、分工、沟通方式 |

### 使用技巧

1. **项目背景写清楚"为什么做"**：比"做什么"更重要，理解了 Why 才能做好 What
2. **决策记录一定要写原因**：不仅记"定了什么"，还要记"为什么这么定"、"讨论过哪些方案"、"为什么否决了其他方案"
3. **风险分等级**：P0/P1/P2，让人一眼知道哪些最紧急
4. **相关人员记清楚**：谁负责什么、找谁对接、什么事找谁，省很多沟通成本

## 预设记忆包框架（可直接填充）

### 00-项目概况（项目管理增强版）

```markdown
# 项目概况

## 项目基本信息
- 项目名称：
- 项目代号：
- 启动时间：
- 预计结束时间：
- 当前阶段：（规划/开发/测试/上线/运维）

## 项目背景
- 为什么要做这个项目？（业务背景、问题、机会）
- 项目目标是什么？（量化指标最好）
- 成功标准是什么？

## 项目范围
- 包含什么：
- 不包含什么：（边界很重要，避免范围蔓延）

## 相关方
- 项目发起人/老板：
- 产品负责人：
- 技术负责人：
- 设计负责人：
- 运营负责人：
- 其他关键角色：

## 项目文档索引
- 产品需求文档：
- 技术方案文档：
- 设计稿：
- 数据看板：
- 其他重要文档：
```

## 常用 Prompt 模板

### 让 AI 接手项目管理

```
我给你一份项目交接包，你先完整读一遍，然后告诉我：
1. 这个项目是做什么的，为什么要做
2. 当前进度如何，下一步该做什么
3. 有哪些风险和问题
4. 有哪些关键决策已经定了

确认你理解了，我们再讨论具体问题。

【附上 IRY 记忆包】
```

### 项目周报生成

```
根据项目当前状态，帮我写一份周报：
- 本周完成了什么
- 当前遇到什么问题
- 下周计划做什么
- 需要什么支持

用简洁的结构化方式输出。

【附上记忆包 + 本周更新】
```

### 风险分析

```
这是我们项目的当前状态和已知风险。帮我分析一下：
1. 哪些风险最紧急，需要立刻处理
2. 哪些风险可能被低估了
3. 还有什么潜在风险我们没考虑到
4. 分别给出应对建议

【附上 03-进度计划.md + 05-已知问题.md】
```

## 项目管理专属 Checklist

交接前确认这些都有了：

- [ ] 项目背景和目标（为什么做、做成什么样算成功）
- [ ] 项目范围（做什么、不做什么）
- [ ] 相关人员和分工（谁负责什么、找谁对接）
- [ ] 当前进度和下一步计划
- [ ] 里程碑和时间节点
- [ ] 待办任务清单（分优先级）
- [ ] 外部依赖（依赖什么、谁负责、什么时候要）
- [ ] 当前问题和风险（分等级）
- [ ] 关键决策记录（定了什么、为什么这么定）
- [ ] 重要文档索引
- [ ] 历史踩坑记录（非常重要！）
