### Key Files to Focus On

| File | Importance | Description |
|------|------------|-------------|
| 00-Project-Overview.md | ⭐⭐⭐⭐⭐ | Project background, goals, scope — first thing the new person reads |
| 03-Progress-Plan.md | ⭐⭐⭐⭐⭐ | Current progress, todos, milestones — core of project management |
| 04-Dependencies-Constraints.md | ⭐⭐⭐⭐ | External dependencies, resource limits, time constraints |
| 05-Known-Issues.md | ⭐⭐⭐⭐ | Risks, problems, pitfalls hit |
| 08-Decision-Log.md | ⭐⭐⭐⭐ | Key decisions, why they were made — avoids rehashing discussions |
| 01-User-Profile.md | ⭐⭐⭐ | Stakeholders, roles, communication styles |

### Pro Tips

1. **Write "why" clearly in project background**: More important than "what" — understanding the Why enables doing the What well
2. **Always include reasons in decision logs**: Not just "what was decided", but "why it was decided this way", "what alternatives were discussed", "why other options were rejected"
3. **Prioritize risks**: P0/P1/P2 — so people can instantly see what's most urgent
4. **Document stakeholders clearly**: Who owns what, who to talk to for what — saves tons of communication cost

## Common Prompt Templates

### Let AI Pick Up Project Management

```
I'm giving you a project handover pack. Read it completely, then tell me:
1. What this project is about and why we're doing it
2. Current progress and what to do next
3. What risks and issues exist
4. What key decisions have already been made

Confirm you understand before we discuss specifics.

[Attach IRY memory pack]
```

### Generate Weekly Project Report

```
Based on the current project status, help me write a weekly report:
- What was completed this week
- What issues were encountered
- What's planned for next week
- What support is needed

Output in concise, structured format.

[Attach memory pack + this week's updates]
```

### Risk Analysis

```
Here's our project's current status and known risks. Help me analyze:
1. Which risks are most urgent and need immediate attention
2. Which risks might be underestimated
3. What potential risks haven't we considered
4. Give mitigation suggestions for each

[Attach 03-Progress-Plan.md + 05-Known-Issues.md]
```

## Project Manager Exclusive Checklist

Make sure these are all covered before handover:

- [ ] Project background and goals (why we're doing it, what success looks like)
- [ ] Project scope (what's in, what's out)
- [ ] Stakeholders and roles (who owns what, who to talk to)
- [ ] Current progress and next steps
- [ ] Milestones and timelines
- [ ] Todo list (prioritized)
- [ ] External dependencies (what's needed, who owns it, when it's due)
- [ ] Current issues and risks (prioritized)
- [ ] Key decision log (what was decided, why)
- [ ] Important document index
- [ ] Historical pitfall records (super important!)
