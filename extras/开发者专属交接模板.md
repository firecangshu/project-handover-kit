## IRY 使用重点

### 必须重点关注的文件

| 文件 | 重要性 | 说明 |
|------|--------|------|
| 02-技术规格.md | ⭐⭐⭐⭐⭐ | 技术栈、架构、核心依赖，开发者最关心 |
| 07-环境配置.md | ⭐⭐⭐⭐⭐ | 怎么跑起来、环境要求、常见坑 |
| 05-已知问题.md | ⭐⭐⭐⭐ | 踩过的坑、bug、技术债务 |
| 08-决策记录.md | ⭐⭐⭐⭐ | 为什么选这个技术栈、为什么这么设计 |
| 06-核心文件索引.md | ⭐⭐⭐⭐ | 代码结构、关键文件位置 |
| 03-进度计划.md | ⭐⭐⭐ | 当前做到哪了、下一步做什么 |

### 使用技巧

1. **扫描前先清理**：把 node_modules、dist、.env 这些无关文件排除，扫描更快更准
2. **重点补全技术决策**：AI 扫描只能看到代码，看不到"为什么这么写"，这部分手动补充
3. **已知问题越详细越好**：踩过的坑、调试过的 bug、尝试过但放弃的方案，都记下来
4. **环境配置写全**：不仅是"怎么运行"，还有"为什么这么配置"、"踩过什么坑"

## 预设记忆包框架（可直接填充）

### 02-技术规格（开发者增强版）

```markdown
# 技术规格

## 技术栈
- 前端：React 18 + TypeScript + Vite
- 后端：Node.js + Express + PostgreSQL
- 部署：Docker + Nginx
- 其他：...

## 核心架构
- 整体架构图（文字描述）
- 核心模块划分
- 数据流向

## 关键依赖
- 核心库及版本
- 为什么选这些库
- 有什么坑需要注意

## 编码规范
- 代码风格（ESLint/Prettier 配置）
- 命名规范
- 目录结构约定
- Git 提交规范
```

## 开发环境

## 常用 Prompt 模板

### 让 AI 接手写代码

```
我给你一份项目交接包，你先完整读一遍，然后告诉我：
1. 这个项目是做什么的
2. 技术栈是什么
3. 当前进度如何
4. 下一步可以做什么

确认你理解了我们再开始干活。

【附上 IRY 记忆包】
```

### Code Review 前先给 AI 上下文

```
这是我们项目的技术规格和编码规范，你先读一下，然后帮我 review 这段代码。

重点看：
1. 有没有符合编码规范
2. 有没有明显的 bug
3. 有没有更好的写法

【附上 02-技术规格.md + 编码规范】
【附上待 review 的代码】
```

### 调试 bug 前先给背景

```
我们项目遇到了一个 bug：【描述 bug 现象】

这是项目的技术背景和之前踩过的坑，你先看一下，然后帮我分析可能是什么原因，怎么排查。

【附上 02-技术规格.md + 05-已知问题.md】
```

## 开发者专属 Checklist

交接前确认这些都有了：

- [ ] 技术栈和版本号
- [ ] 核心架构说明
- [ ] 环境配置步骤（确保新人能跑起来）
- [ ] 关键依赖及注意事项
- [ ] 已知 bug 和临时解决方法
- [ ] 技术债务清单
- [ ] 踩坑记录（非常重要！）
- [ ] 编码规范
- [ ] Git 工作流
- [ ] 部署流程
